Objetivo do Projeto

Criar uma aplicação web que usa a câmara do dispositivo para reconhecer gestos da Língua Gestual Portuguesa (LGP) e os traduz para texto em tempo real. O modelo de reconhecimento é treinado pelos próprios alunos no browser. A aplicação inclui síntese de voz integrada, construtor de frases interativo e um servidor Python (Flask) para guardar e gerir de forma permanente o histórico de tradução.

Distribuição de 15 horas
F1
Descoberta e Contexto 2h

Antes de programar, perceber o propósito. A LGP é uma língua completa com gramática própria — não se baseia apenas em soletrar letras.

  • Ver um vídeo curto explicativo sobre a LGP e a comunidade surda em Portugal
  • Conhecer o alfabeto manual português (dactilologia) — as 26 letras e os dígitos
  • Decidir em grupo o subconjunto a traduzir: as letras A–Z (dactilologia) ou 10 palavras simples
  • Criar uma conta Google (necessária para utilizar a Teachable Machine)
  • Abrir o Teachable Machine pela primeira vez e explorar a interface
💡 Recomendação: Começar com apenas 5 letras (A, B, C, D, E) nas primeiras sessões. Um modelo com poucas classes bem treinadas é melhor e mais preciso do que com 26 mal treinadas.
F2
Recolha de Dados e Treino do Modelo 3h

Esta é a fase crítica. Um modelo de Machine Learning só é eficaz com dados diversificados. Cada participante contribui com capturas dos seus próprios gestos.

  • Aceder a teachablemachine.withgoogle.com → New Project → Image Project → Standard
  • Criar uma classe específica por letra/palavra (ex: "A", "B", "C", "Olá", "Obrigado")
  • Para cada classe: recolher **no mínimo 50 fotografias** com a câmara do browser
  • Variar fatores: focar em fundos diferentes, iluminação diferente, ângulos de mão e com as duas mãos
  • Adicionar uma classe "Nenhum" com imagens neutras sem gestos (essencial para reduzir falsos positivos)
  • Clicar em Train Model e aguardar (geralmente demora entre 2 a 5 minutos)
  • Testar exaustivamente no preview da plataforma antes de avançar para a exportação
⚠️ Erro comum: Capturar fotografias sempre no mesmo plano de fundo ou na mesma posição. O modelo ficará dependente desse ambiente. Diversifique!
💡 Dataset extra: Podem ser recolhidas e adicionadas imagens de alfabetos manuais já existentes para reforçar o modelo. Consulte a Tab "Guia" para recursos adicionais.
F3
Exportar Modelo e Ligar à App 2h

A Teachable Machine gera uma hiperligação para o modelo alojado. A aplicação carrega esse link e inicia a deteção em tempo real.

  • Na plataforma: Export Model → Tensorflow.js → Upload (shareable link)
  • Copiar o URL gerado (exemplo: https://teachablemachine.withgoogle.com/models/yRu8axD3u/)
  • Ir ao painel do "Tradutor" nesta app
  • Colar o endereço copiado no formulário e premir "Carregar Modelo"
  • Após a validação e confirmação, premir "Iniciar Câmara" para testar as predições
💡 O link é o seu modelo: Pode partilhar o link gerado com colegas. Qualquer pessoa com a app pode utilizá-lo sem ter de repetir o processo de treino.
F4
Melhorar e Personalizar 5h

Com as deteções básicas a funcionar, aprofunde as potencialidades da aplicação.

  • Adicionar novas palavras ou expressões ao modelo, procedendo a um novo treino e exportação
  • Personalizar o estilo visual, cores do tema e títulos na interface
  • Construir um histórico de deteções interativo para recolher e formar frases completas
  • Testar com novos utilizadores que não integraram a base de recolha inicial para detetar falhas
  • Ajustar o limiar de sensibilidade (Threshold) de modo a mitigar ruídos ou falsas deteções
F5
Publicar e Apresentar 3h

Partilhar o seu tradutor publicamente online e apresentá-lo.

  • Publicar os ficheiros estáticos gratuitamente via GitHub Pages (arrastando os ficheiros para o repositório)
  • Testar no telemóvel: guardar o link no ecrã principal como PWA (Progressive Web App)
  • Preparar uma breve apresentação de 3 minutos cobrindo: Problema → Solução → Demonstração ao vivo
  • Fazer uma reflexão em grupo sobre melhorias futuras e o potencial impacto na inclusão social

Criação de Modelos Passo a Passo
  • Abra a plataforma teachablemachine.withgoogle.com no seu navegador (preferencialmente Google Chrome).
  • Selecione "Get Started" → "Image Project" → "Standard image model".
  • Renomeie as classes para representar a letra ou gesto pretendido (ex: clique em "Class 1" e digite "A").
  • Prima o botão "Webcam" e mantenha pressionado o botão de captura para tirar imagens dos gestos (mínimo 50 por classe).
  • Adicione obrigatoriamente uma classe com o nome "Nenhum" e capture fotos neutras (apenas do fundo ou corpo relaxado).
  • Concluídas as classes, clique no botão central "Train Model".
  • Teste o desempenho do reconhecimento no painel de "Preview" à direita.
  • Estando funcional: selecione "Export Model" → "Tensorflow.js" → "Upload (shareable link)" e copie a hiperligação gerada.
  • Insira esse link no painel "Tradutor" desta aplicação e clique em carregar.

Dica: comece por criar classes para as primeiras 5 letras do abecedário (A, B, C, D e E).

Recomendações para Obter Maior Precisão
  • Volume de dados: Tente obter no mínimo 50 capturas por gesto. Quanto maior for o conjunto de dados, melhor será a deteção.
  • Variação de cenários: Registe fotos alterando a distância do ecrã, a iluminação e os planos de fundo.
  • Diversificação física: Registe gestos com a mão esquerda e direita, alterando ligeiramente a inclinação dos dedos.
  • Trabalho cooperativo: Ao partilhar amostras de treino de várias pessoas, o modelo adquire uma robustez de deteção superior.
  • A importância do "Nenhum": Esta classe evita que o modelo tente adivinhar gestos quando não está ninguém à frente da câmara.
